
# Лабораторная работа №3  
## Разработка простой темы WordPress  

## Цель работы  
Целью данной лабораторной работы является изучение процесса создания простой пользовательской темы WordPress, понимание минимальной структуры темы и принципов работы шаблонов WordPress.

## Используемые инструменты  
В процессе выполнения лабораторной работы были использованы следующие инструменты:

- XAMPP для запуска локального сервера;
- WordPress, установленный в локальной среде;
- Visual Studio Code для создания и редактирования файлов темы;
- браузер Google Chrome для проверки результата.

## Ход выполнения работы  

### 1. Подготовка среды  
В локально установленном WordPress был открыт каталог `wp-content/themes`. Внутри него была создана новая папка пользовательской темы с именем `usm-theme`. После этого в файле `wp-config.php` был включён режим отладки с помощью строки:

```php
define('WP_DEBUG', true);
````

### 2. Создание обязательных файлов темы

Для корректной работы темы WordPress обязательными являются как минимум два файла: `style.css` и `index.php`.

Сначала в папке `usm-theme` был создан файл `style.css`. В него были добавлены метаданные темы, необходимые для распознавания темы системой WordPress:

```css
/*
Theme Name: USM Theme
Author: Marina Glavceva
Description: Custom theme for laboratory work
Version: 1.0
*/
```

После метаданных в файл были добавлены базовые CSS-стили для оформления основных элементов страницы.

Далее был создан файл `index.php`, который является основным шаблоном темы. В этом файле была реализована базовая структура вывода записей с использованием стандартного цикла WordPress (Loop).


### 3. Создание общих компонентов шаблонов

Для удобства и правильной организации кода были созданы отдельные файлы общих частей шаблонов:

* `header.php` — для верхней части страницы;
* `footer.php` — для нижней части страницы.

В файле `header.php` была размещена начальная HTML-структура документа, тег `<head>`, вызов функции `wp_head()` и заголовок сайта.
В файле `footer.php` был размещён подвал сайта и вызов функции `wp_footer()`.

Подключение этих частей в `index.php` было выполнено с помощью функций:

```php
<?php get_header(); ?>
<?php get_footer(); ?>
```

Также на главной странице был настроен вывод последних записей сайта с помощью WordPress Loop.


### 4. Создание файла functions.php

В папке темы был создан файл `functions.php`. Этот файл используется для добавления функциональности темы.

Внутри `functions.php` была добавлена функция для подключения основного файла стилей темы:

```php
<?php
function usm_theme_styles() {
    wp_enqueue_style('main-style', get_stylesheet_uri());
}
add_action('wp_enqueue_scripts', 'usm_theme_styles');
```

Таким образом, стили из `style.css` начали корректно подключаться на страницах сайта.


### 5. Создание дополнительных шаблонов

Для более полной структуры темы были созданы дополнительные файлы шаблонов:

* `single.php` — для отображения отдельной записи;
* `page.php` — для отображения страниц;
* `sidebar.php` — для боковой панели;
* `comments.php` — для отображения комментариев;
* `archive.php` — для отображения архивов записей.

Файл `sidebar.php` был подключён в соответствующие шаблоны с помощью функции:

```php
<?php get_sidebar(); ?>
```

Файл `comments.php` был подключён в шаблоны `single.php` и `page.php` с помощью функции:

```php
<?php comments_template(); ?>
```

Это позволило разделить структуру темы на логические части и приблизить её к реальной структуре WordPress-темы.

### 6. Стилизация темы

После создания основных файлов темы были добавлены стили для следующих элементов:

* заголовка сайта;
* содержимого страницы;
* записей;
* подвала сайта;
* боковой панели.

В результате тема приобрела более аккуратный и понятный внешний вид. На главной странице записи стали отображаться в виде карточек с заголовками, текстом и изображениями.

### 7. Добавление скриншота темы

В папку темы был добавлен файл `screenshot.png` размером `1200x900` пикселей. Этот файл используется WordPress для отображения превью темы в панели администратора в разделе «Внешний вид → Темы».

После добавления `screenshot.png` тема стала отображаться с изображением предпросмотра.


### 8. Активация темы

После создания всех основных файлов тема была активирована в административной панели WordPress через раздел:

`Внешний вид → Темы`

После активации темы сайт начал использовать созданные шаблоны и стили. На главной странице корректно отобразились записи, меню сайта и другие элементы интерфейса.

## Ответы на контрольные вопросы

### 1. Какие два файла обязательны для любой темы WordPress?

Для любой темы WordPress обязательными являются файлы `style.css` и `index.php`.
Файл `style.css` содержит метаданные темы и стили, а `index.php` является основным шаблоном для отображения контента.

### 2. Как подключаются общие части шаблонов?

Общие части шаблонов подключаются с помощью специальных функций WordPress:

* `get_header()` — подключает `header.php`;
* `get_footer()` — подключает `footer.php`;
* `get_sidebar()` — подключает `sidebar.php`.

### 3. В чём разница между index.php, single.php и page.php?

`index.php` — это основной шаблон темы, который используется по умолчанию.
`single.php` предназначен для отображения отдельной записи блога.
`page.php` используется для отображения отдельных страниц сайта.

### 4. Какова роль файла functions.php в теме?

Файл `functions.php` используется для добавления функциональности темы.
Через него можно подключать стили и скрипты, регистрировать меню, виджеты, области sidebar и добавлять различные возможности темы.
